1 Project Overview & Motivation

What Is Smart Health Analyzer?

Smart Health Analyzer is a full-stack web application that helps users monitor, track, and improve their health through multiple integrated modules — calorie tracking, water intake, mental health screening, AI-powered meal planning, and streak-based habit gamification.

Why Was This Built?

ProblemOur Solution
Users need 5+ apps to track food, water, mental health, exercise, and caloriesOne unified platform with 9 integrated modules
Many health apps show fake "estimated" values for blood pressure, cholesterolOnly scientifically validated formulas — no fake estimates
Personal dietitian consultations are expensive (₹500–₹2,000/session)Free AI-powered meal planning and calorie guidance
70% of Indians with mental health symptoms never seek professional screeningPrivate, stigma-free screening using standard clinical tools
Users start tracking but stop after a few daysGamified streaks and badges for motivation

Target Users

  • Health-conscious individuals wanting to track daily habits
  • People starting a fitness or weight management journey
  • Anyone curious about their mental health but hesitant to visit a professional first

4 System Architecture

⬇                  ⬇

Data Flow

  1. User interacts with React UI (clicks button, fills form)
  2. React calls api.js → Axios sends HTTP request to FastAPI backend
  3. FastAPI validates request using Pydantic models
  4. Backend queries/inserts data in Supabase (PostgreSQL)
  5. For AI features: Backend calls NVIDIA API (primary) or Gemini (fallback)
  6. Response flows back: Backend → Axios → React state → UI re-renders

5 Module 1 — Health Check & BMI

Two-step questionnaire: personal info (age, gender, height, weight) followed by lifestyle habits (smoking, exercise, diet, stress). The only metric displayed to users is BMI — because it's the only value that's 100% accurate from user input.

Body Mass Index (BMI)
BMI = Weight (kg) ÷ Height (m)²

Worked Example

Weight = 70 kg, Height = 170 cm
Height in meters = 1.70
BMI = 70 ÷ (1.70)² = 70 ÷ 2.89 = 24.2 → Normal
BMI RangeCategoryColor Indicator
< 18.5Underweight🟡 Yellow
18.5 – 24.9Normal🟢 Green
25.0 – 29.9Overweight🟡 Yellow
≥ 30.0Obese🔴 Red
✅ Accuracy Note BMI is the international standard formula used by WHO, NHS, and hospitals worldwide. The calculation uses only user-provided measurements — no estimation involved.
⚠️ Design Decision We originally displayed estimated blood pressure, cholesterol, and glucose values based on lifestyle answers. These were removed because they were NOT real measurements and could mislead users into skipping actual medical tests.

6 Module 2 — Calorie Calculator

Uses the Mifflin-St Jeor equation (1990) — the most accurate clinically validated formula for estimating Basal Metabolic Rate (BMR). All calculations run client-side in real-time.

Basal Metabolic Rate (BMR) — Mifflin-St Jeor Equation
Men: BMR = (10 × W) + (6.25 × H) - (5 × A) + 5
Women: BMR = (10 × W) + (6.25 × H) - (5 × A) - 161

W = weight in kg, H = height in cm, A = age in years

What Is BMR?

The number of calories your body burns at complete rest — just to keep organs functioning, breathing, and maintaining body temperature. This is the minimum energy you need even if you lay in bed all day.

Total Daily Energy Expenditure (TDEE)
TDEE = BMR × Activity Factor
Activity LevelFactorDescription
Sedentary1.200Little/no exercise, desk job
Lightly Active1.375Light exercise 1-3 days/week
Moderately Active1.550Moderate exercise 3-5 days/week
Very Active1.725Hard exercise 6-7 days/week
Extremely Active1.900Athlete or physical labor job

Goal-Based Calorie Adjustment

GoalAdjustmentExpected Result
Lose WeightTDEE − 500 kcal~0.5 kg/week loss
MaintainTDEE (no change)Stay at current weight
Gain WeightTDEE + 500 kcal~0.5 kg/week gain

Why 500 kcal? Because 1 kg of body fat ≈ 7,700 kcal. A daily deficit of 500 × 7 days = 3,500 kcal/week ≈ 0.45 kg/week — the medically recommended safe rate.

Worked Example

// 25-year-old male, 70 kg, 170 cm, Moderately Active, Goal: Lose
BMR = (10 × 70) + (6.25 × 170) - (5 × 25) + 5
    = 700 + 1062.5 - 125 + 5
    = 1,642 kcal/day

TDEE = 1,642 × 1.55 = 2,545 kcal/day
Target = 2,545 - 500 = 2,045 kcal/day

Macro Nutrient Breakdown

Protein (g) = (Calories × Ratio) ÷ 4 cal/g
Carbs (g) = (Calories × Ratio) ÷ 4 cal/g
Fat (g) = (Calories × Ratio) ÷ 9 cal/g
GoalProteinCarbsFatRationale
Lose Weight35%40%25%Higher protein preserves muscle during deficit
Maintain30%45%25%Balanced for general health

7 Module 3 — Mental Health Screening

Uses two internationally standardized clinical screening tools: PHQ-9 for depression and GAD-7 for anxiety. These are the same instruments used by doctors in hospitals and clinics worldwide.

Scoring

Each answer: 0–3, Range: 0–27

📋 Clinical Note Both PHQ-9 and GAD-7 are validated screening tools — NOT estimates. The scoring directly reflects a user's answers using the same cutoffs used in clinical settings. When scores indicate severe symptoms, the app displays: "Please consider speaking with a mental health professional."

8 Module 4 — Food Logger

Features

  • 4 meal categories: Breakfast, Lunch, Dinner, Snack
  • Track per item: food name, calories, protein, carbs, fat, quantity, unit
  • Daily calorie ring (consumed vs 2,000 kcal goal)
  • Expandable meal sections with individual food items
  • Direct integration with AI Meals (one-click "Add to Food Log")

Technical Flow

// Adding food:
User fills form → POST /nutrition/log
→ Pydantic validates → calories cast to int()
→ Supabase INSERT → Streak auto-updated
→ GET /nutrition/logs → UI refreshes with new data

// Fetching daily summary:
GET /nutrition/summary?user_id=xxx
→ Backend aggregates from nutrition_logs table
→ Returns: total_calories, total_protein, total_carbs, total_fat

Database Columns

id, user_id, meal_type, food_name, quantity, unit, calories (INTEGER), protein, carbs, fat, fiber, meal_time, date, logged_at, log_date

9 Module 5 — Water Tracker

Features

  • Glass-by-glass tracking (1 glass = 250 ml)
  • SVG ring progress indicator with glow effects
  • Customizable daily goal (default: 8 glasses = 2,000 ml)
  • Weekly bar chart history
  • Streak counter for consecutive goal-hitting days
Total (ml) = glasses × 250
Progress % = (glasses ÷ goal) × 100
Ring Offset = circumference × (1 − progress/100)

14 Authentication & Security

  • 🔒 Password Hashing — bcrypt via Supabase Auth
  • 🔒 JWT Tokens — Time-limited session tokens
  • 🔒 CORS — Only allowed origins can access the API
  • 🔒 Row Level Security — Database-level user data isolation
  • 🔒 Service Key — Backend-only, never exposed to frontend
  • 🔒 Pydantic Validation — Rejects malformed requests

15 Technical Challenges & Solutions

Bug 1: Food Logger "Add" Button — Data Written but Never Displayed

Problem: .order("created_at") used in GET query, but actual DB column is logged_at. Every read query returned HTTP 400.

Fix: Changed to .order("logged_at", desc=False)

Bug 2: "Invalid input syntax for type integer: '206.0'"

Problem: Frontend sent calories: 206.0 (float) to an INTEGER column in PostgreSQL.

Fix: Added int() cast in backend + Math.round() in frontend.

Bug 3: Health Check Buttons Pre-Selected on Load

Problem: Number('') evaluates to 0 in JavaScript, matching the first button option (value 0).

Fix: Added formData[field] !== '' check before comparing.

Bug 4: Pydantic Rejecting Requests with Extra Fields

Problem: Frontend sent meal_time field not defined in the Pydantic model.

Fix: Added model_config = ConfigDict(extra="ignore").

Bug 5: Streaks Reading Wrong Properties

Problem: Frontend read streakData.water_streak but backend returns streakData.streaks.water.current.

Fix: Corrected all property access paths to match backend response.

Design Decision: Removing Estimated Medical Values

Issue: Displaying "Estimated BP: 128/82" based on lifestyle guesses could mislead users.

Decision: Removed ALL estimated values. Only scientifically accurate calculations are shown.
